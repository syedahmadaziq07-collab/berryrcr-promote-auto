Fix "Add Group Manual" ID normalization bug.

PROBLEM:
User inputs Telegram supergroup ID: -1001156883698
Userbot has already joined this group.
It's a normal supergroup (not channel).
User can manually send messages in the group.
But bot replies: "❌ Kumpulan tidak dijumpai."

ROOT CAUSE:
Code compares raw Telegram ID (-1001156883698) with Telethon entity.id directly.
Telethon uses internal ID format without -100 prefix for supergroups.

Example mismatch:
- Telegram ID: -1001156883698
- Telethon entity.id: 1156883698
- They don't match → "not found"

2. TELEGRAM ID FORMAT REFERENCE:

   Format types:
   - Personal chat: 123456789 (positive integer)
   - Bot: 987654321 (positive integer)
   - Group (legacy): -123456789 (negative, no prefix)
   - Supergroup: -1001234567890 (negative with -100 prefix)
   - Channel: -1001234567890 (same as supergroup)
   - Broadcast channel: -1001234567890
   
   Telethon internal format:
   - Supergroup/Channel: 1234567890 (removes -100 prefix, positive)
   - Group (legacy): -123456789 (keeps negative)
   - Personal: 123456789 (unchanged)

4. UPDATE ADD GROUP MANUAL HANDLER:

   Current logic (WRONG):
```python
   user_input_id = int(message.text)  # e.g. -1001156883698
   
   # Scan dialogs
   async for dialog in client.iter_dialogs():
       if dialog.id == user_input_id:  # ❌ NEVER MATCHES
           # save group
   
   # Not found
   await message.reply("❌ Kumpulan tidak dijumpai.")
```
   
   Fixed logic (CORRECT):
```python
   from bot.utils.id_normalizer import normalize_telegram_id
   
   telegram_id, telethon_id = normalize_telegram_id(message.text)
   
   found = None
   async for dialog in client.iter_dialogs():
       # Compare with BOTH formats
       if dialog.id == telegram_id or dialog.id == telethon_id:
           found = dialog
           break
   
   if found:
       # Save using TELEGRAM format (the one user sees)
       await save_group(user_id, telegram_id, found.title)
       await message.reply(f"✅ Berjaya tambah: {found.title}")
   else:
       await message.reply("❌ Kumpulan tidak dijumpai atau bot belum join.")
```
