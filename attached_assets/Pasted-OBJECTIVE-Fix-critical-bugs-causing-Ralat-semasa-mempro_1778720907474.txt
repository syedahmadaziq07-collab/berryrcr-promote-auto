FIX #2: Fix Transaction Order in plan_flow.py
Location: bot/handlers/plan_flow.py around line 214-258
PROBLEM:
Coins are deducted BEFORE subscription is created. If subscription creation fails, user loses coins but gets no plan.
CURRENT CODE (WRONG):
python# Line ~214
ok = await db.deduct_coins(uid, total, ...)  # ← Coins DELETED first ❌

# Line ~258  
started, expires = await db.create_subscription(uid, plan_key, months)  # ← If this CRASHES...
# Result: User lost coins but no subscription! 💀
FIX TO:
python# STEP 1: Create subscription FIRST
started, expires = await db.create_subscription(uid, plan_key, months)

# STEP 2: Only deduct coins if subscription was successful
if started:
    ok = await db.deduct_coins(uid, total, f"Beli plan {plan_key.upper()} ({months} bulan)")
    if not ok:
        # If deduct fails, rollback subscription (optional - add rollback function)
        await callback.message.edit_text("❌ Ralat menolak syiling. Sila cuba lagi.")
        return
else:
    await callback.message.edit_text("❌ Gagal membuat subscription. Sila cuba lagi.")
    return
